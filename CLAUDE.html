CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

What this is

A personal Chinese-language blog (Randy's Blog) built with VuePress 2 (rc.18) + vuepress-theme-reco (rc.25) using the Vite bundler. Output is deployed as a static site to wang-haihua.github.io.

Commands

npm install        # install deps
npm run dev        # vuepress dev .  — local preview at http://localhost:8080
npm run build      # vuepress build . — outputs to ./public (per dest in config.ts)
npm run deploy     # bash scripts/deploy.sh — build, then force-push public/ to wang-haihua.github.io
npm run push       # bash scripts/push.sh   — quick `git add . && commit -m 'feat: add doc' && push origin main` for source

There is no test suite (npm test is a stub).

Note: scripts/deploy.sh does cd ../public after build, so it expects to be run from inside scripts/ (or relies on the npm-script CWD quirk). If editing, be aware it git inits inside public/, force-pushes to the wang-haihua.github.io.git repo's main branch, then deletes public/.

Layout

.vuepress/
  config.ts          # site + theme config (navbar, socialLinks, footer text, lang)
  client.ts          # registers custom components — currently @vicons icon components
  components/        # Vue SFCs auto-available in markdown
  styles/index.css   # global CSS (footer overrides, hero tweaks, dark-mode rules)
  public/            # static assets served at site root
    picture/<category>/<year>/<post-id>/...   # post images live here
blogs/               # post sources, grouped by category folder, then year folder
  bigdata/2024/...md
  thinking/2026/...md
  personalInfo/README.md
scripts/
  deploy.sh          # build + publish to GitHub Pages repo
  push.sh            # commit + push source

.vuepress/.cache/ and .vuepress/.temp/ are build caches. public/ is the build output (gitignored).

Post conventions

Posts are markdown under blogs/<category>/[<year>/]*.md with YAML frontmatter.

Detailed rules for post writing live in .claude/rules/post-writing.md — frontmatter schema, heading numbering (01 / 1.1 / 1.1.1, max 3 levels), Chinese punctuation rule (no trailing 。), and image reference convention. That file auto-loads when Claude touches blogs/**/*.md.

Theme customization touchpoints

  • Navbar / social links / i18n text → .vuepress/config.ts (recoTheme({...})).
  • Footer is double-customized: a <script> injected via head in config.ts hides the theme-info span on DOMContentLoaded, and .vuepress/styles/index.css enforces the same with !important. Edits to footer styling must keep both in sync.
  • New icons in social links must be registered in .vuepress/client.ts via app.component('Name', IconFromVicons) before referencing icon: 'Name' in config.ts.
  • Homepage hero/buttons → README.md at repo root (it's the home page's frontmatter, with home: true and bannerBrand).

References

  • vuepress-reco v2 docs: https://theme-reco.vuejs.press/docs/guide/introduce.html
  • @vicons icon packs (used here: @vicons/ionicons5, @vicons/antd): https://github.com/07akioni/xicons
